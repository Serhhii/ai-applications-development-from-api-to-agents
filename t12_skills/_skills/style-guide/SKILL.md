# Style Guide Skill

## File Map

| File | Purpose |
|---|---|
| `REFERENCE.md` | Type-specific rules per content type (emails, blog, social, support) |
| `EXAMPLES.md` | Before/after rewrites showing the style in action |

Always read `REFERENCE.md` for the relevant content type before rewriting. Use `EXAMPLES.md` as a pattern reference.

## Workflow

1. **Identify** the content type: email, blog, social media, or support reply.
2. **Read** `REFERENCE.md` — find the section for that content type.
3. **Apply** the Core Principles below to the text.
4. **Check** `EXAMPLES.md` for a matching before/after pattern.
5. **Return** the polished text followed by a short **Changes:** list.

## Core Principles

- **Voice:** Human, direct, confident — we talk *with* people, not *at* them.
- **Tone:** Warm but professional. Never stiff, never sloppy.
- **Length:** Shorter is better. Cut every word that adds no meaning.
- **Person:** Address the reader as "you". Use "we" for the company.
- **Active voice:** Prefer active constructions. Passive is a last resort.
- **Contractions:** Use them (it's, we're, you'll) — they sound natural.

## Instant Rules

- ❌ No jargon or corporate-speak ("leverage", "synergy", "going forward")
- ❌ No passive voice where active is possible
- ✅ Contractions are fine
- ✅ Numbers under 10: spell out; 10 and above: use digits
- ✅ Oxford comma always
- ❌ No more than one exclamation mark per piece
- ✅ Sentence case for headings, not Title Case
